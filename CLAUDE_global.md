## Project

**Yalla.House** — a real-estate infrastructure platform. First market: **Germany (DE)**. Owners sell/rent without agents via portal aggregation (ImmobilienScout24, Immowelt). Full-stack monorepo.

---

## Monorepo Structure

```
Yalla.House/
  apps/web/           ← Next.js 14 (App Router) — primary application
  packages/db/        ← PostgreSQL schema + seed (Supabase)
  packages/integrations/ ← Portal connectors (IS24, Immowelt, ...)
  packages/ui/        ← Shared React UI components (planned)
  Website/            ← Original static HTML/CSS/JS (design reference, not built)
  _archive/           ← Archived experiments (LandingPage, old travel site)
```

Package manager: **pnpm**. Build orchestration: **Turborepo**.

> Node.js must be installed first: `brew install node@20 && npm install -g pnpm`

---

## Commands

```bash
# From repo root
pnpm install              # Install all workspace dependencies
pnpm dev                  # Start all apps in dev mode (turbo)
pnpm build                # Build all packages + apps
pnpm db:push              # Push schema.sql to Supabase
pnpm db:generate-types    # Regenerate packages/db/src/types.ts from live schema

# From apps/web/
pnpm dev                  # Next.js dev server (localhost:3000)
pnpm type-check           # TypeScript check without emit
pnpm lint                 # ESLint via next lint

# From packages/integrations/
pnpm type-check           # TypeScript check
```

## Key Architecture

### apps/web (Next.js 14 App Router)

- **Routing**: `src/app/[locale]/` — all user-facing routes are locale-prefixed
- **Default locale**: `de` (served at `/`, `/owner`, etc. without prefix — `as-needed` strategy)
- **Auth**: Supabase magic link (OTP). Session managed via `@supabase/ssr`. Auth callback at `/api/auth/callback`.
- **Middleware**: `src/middleware.ts` — applies i18n routing + Supabase session refresh + redirect on unauthenticated access to `/owner`, `/hunter`, `/agent`, `/admin`
- **Supabase clients**:
  - `src/lib/supabase/client.ts` — browser client (Client Components)
  - `src/lib/supabase/server.ts` — server client (Server Components, API routes) + `createServiceClient()` for admin/Inngest
- **Background jobs**: Inngest at `/api/inngest`. Add new functions in `src/inngest/functions/` and register in `src/app/api/inngest/route.ts`
- **i18n**: `next-intl`. Messages in `messages/de.json` and `messages/en.json`
- **Styling**: Tailwind CSS. Brand tokens: `bg-brand` (#FFD400), `bg-bg` (#EDEEF2), `bg-surface` (#FFFFFF). Font: Plus Jakarta Sans. `rounded-card` = 16px.

### packages/db

- `schema.sql` — full Supabase/PostgreSQL schema. **DE-first**: German-specific columns on `listings` (`title_de`, `description_de`, `nebenkosten`, `kaution`, `floor`, `total_floors`, `construction_year`, `energy_class`). `place_id` auto-generated by trigger (`yh_de_abc123`).
- `seed.sql` — `country_config` DE is `is_active=true`; GB/AT/CH inactive. Portal configs: `immoscout24` + `immowelt` active. Subscription plans: Basis €199, Professional €499, Komplett €899.
- All prices stored as **integer minor units** (cents). `fromMinorUnits()` converts for display/portal API.

### packages/integrations

- `src/portals/types.ts` — `PortalConnector` interface + shared types (`Listing`, `ValidationResult`, etc.)
- `src/portals/immoscout24.ts` — IS24 connector: OAuth2 client_credentials, sandbox support, full validate/transform/submit/update/withdraw/getStatus. IS24 creates listings as INACTIVE → separate PUT to publish on channel 10000.
- `src/portals/immowelt.ts` — Immowelt connector: API key (`x-api-key`), same method set. Auto-syndicates to immonet.de.
- `src/portals/registry.ts` — `connectorRegistry` singleton. `registry.get('immoscout24')` returns the connector.

### Feed Export Flow (Inngest)

`feed/export.requested` event → `feedExport` function:
1. Fetch listing + portal config + field mappings + credentials
2. `connector.validateListing()` → if invalid, write `failed` to `listing_portal_status`
3. `connector.transformListing()` → portal-specific payload
4. If `external_id` exists → `updateListing`, else → `submitListing`
5. Upsert `listing_portal_status` with result

## Data Conventions

- **Currency**: always stored as `int` minor units (cents/Rappen). Use `fromMinorUnits(value, currency)` from `@yalla/integrations` for display.
- **IDs**: `uuid` everywhere. `place_id` is the stable public identifier (e.g. `yh_de_abc123`), never changes.
- **Timestamps**: `timestamptz` in UTC.
- **Country fields**: German-specific fields are top-level columns on `listings` (not in `country_fields` JSON) for query performance. `country_fields jsonb` holds anything else.
- **Referral**: `UNIQUE(referrer_id, referred_user_id)` — first referrer wins. Self-referral blocked by DB trigger.

## Environment Variables

See `.env.example` at repo root. Required for `apps/web`:
- `NEXT_PUBLIC_SUPABASE_URL` + `NEXT_PUBLIC_SUPABASE_ANON_KEY` + `SUPABASE_SERVICE_ROLE_KEY`
- `IMMOSCOUT24_CLIENT_ID` + `IMMOSCOUT24_CLIENT_SECRET` (+ `IMMOSCOUT24_SANDBOX=true` for dev)
- `IMMOWELT_API_KEY` (+ `IMMOWELT_SANDBOX=true` for dev)
- `INNGEST_SIGNING_KEY` + `INNGEST_EVENT_KEY`
- `STRIPE_SECRET_KEY` + `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` + `STRIPE_WEBHOOK_SECRET`

## Adding a New Portal Connector

1. Create `packages/integrations/src/portals/<slug>.ts` implementing `PortalConnector`
2. Register in `packages/integrations/src/portals/registry.ts`
3. Export from `packages/integrations/src/index.ts`
4. Add portal row to `packages/db/seed.sql` and run `pnpm db:push`
5. Add portal field mappings to `seed.sql`

## Website/ (Static Reference)

The `Website/` directory contains the original static HTML/CSS/JS. It is **not** part of the monorepo build — it's a design reference. Do not modify it when working on the Next.js app. The design tokens (brand yellow #FFD400, bg #EDEEF2, surface white, Plus Jakarta Sans) have been ported to `apps/web/tailwind.config.ts`.

The `LandingPage/` directory (Vite + React prototype) has been archived to `_archive/LandingPage/` — it was a standalone experiment not connected to the monorepo build.
