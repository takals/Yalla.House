Conversational Avatar Intake

Replace every form in Yalla.House with a guided conversation. Users talk, the brief builds itself, and the AI learns to ask better questions over time.

The Solution

Talk to Yalla. Watch your brief build.

A conversational AI assistant that asks questions one at a time, transcribes answers in real time, and lets users edit inline. The brief panel shows every answer as it lands — visible, editable, theirs.

💬

Natural Conversation

Questions flow like a chat, not a form. "Tell me about the property — what makes it special?" instead of "Description (max 500 chars)".

🎙️

Voice or Text

Users choose how they answer. Voice for speed, text for precision. The mic button activates browser-native speech recognition — zero API cost.

📝

Live Brief Panel

Every answer populates a structured field in real time. Users see their brief building on screen. Click any field to edit — full control, zero friction.

🧠

Gets Smarter Over Time

The avatar remembers returning users, pre-fills known data, and optimises question order based on which conversations produce the best outcomes.

🌍

Multilingual

Speaks English and German natively. Detects language from user input and switches automatically. Web Speech API supports 100+ languages.

🔒

Privacy-First

Voice is processed locally in the browser — never sent to external servers. The transcript lives on the user's device until they choose to submit.

Intelligence Layer

The avatar gets smarter with every conversation

Three layers of learning compound over time, creating a moat that deepens with every user interaction.

01

Per-User Memory

Returning users get pre-filled data. "Welcome back, Tarek — same address or a different property?" Second listing takes 60 seconds instead of 5 minutes. The avatar remembers communication style, preferred language, and previous answers.

02

Cross-Platform Learning

Analyses which question sequences produce the best briefs. If detailed seller situations generate 3x more agent responses, the avatar learns to probe deeper there. Optimises flow based on outcomes, not just completion.

03

Market Intelligence

Adapts to local terminology and norms. In East London, "ex-council" is common. In Munich, it asks about Energieausweis. Learns pricing patterns, popular property types, and seasonal trends per postcode.

Where It Applies

Every data intake becomes a conversation

Owner

Property Brief Creation

Instead of 30+ form fields, owners answer 9 natural questions. The avatar extracts address, type, beds, baths, size, price, description, situation, and timeline — building the Owner Brief that agents compete over.

Hunter

Home Passport Setup

"What are you looking for? What's your budget? Any dealbreakers?" Three minutes of conversation replaces a dry profile form. The passport powers agent matching and property filtering.

Agent

Profile & Coverage Setup

"What's your agency? What areas do you cover — sale, rental, or both?" The avatar maps spoken postcodes to structured coverage areas. Agents hate admin — this removes it.

Agent

Brief Response

When agents receive a brief, they can respond via conversation too: "I'd charge a 1% fee, here are three properties I'd suggest, and here's why I'm the right fit." Natural proposals, not form fields.

All Roles

Support & Guidance

The avatar doubles as contextual help. "What does 'advisory tier' mean?" — instant explanation within the flow. No help docs, no FAQ page, no leaving the conversation.

Technical Architecture

Built on browser-native APIs

The core runs at zero marginal cost per conversation. Voice processing happens locally, structured extraction uses LLM, and the learning layer trains on outcomes.

🎤

Web Speech API

Browser-native speech-to-text. No API cost. Supports 100+ languages. Real-time transcription with interim results. Falls back to text input gracefully.

⚡

Conversational Engine

Lightweight state machine defines question flow. LLM extracts structured fields from natural language answers. Validates and confirms before writing to Supabase.

🔄

Learning Pipeline

Conversation logs → outcome tracking (did the brief generate responses?) → question flow optimisation. Runs as an Inngest background job. No real-time ML inference needed.

Rollout Plan

Three phases, each self-contained

Each phase delivers value independently. We don't need voice to ship Phase 1 — text conversation alone is already a 10x UX improvement over forms.

Phase 1 — Text Chat

Sprint Zero (Week 1-2)
  • Chat-style question flow for Owner Brief
  • Live brief panel with inline editing
  • Quick-select chips for structured answers
  • Postcode-based agent preview
  • Extend to Hunter Passport intake

Phase 2 — Voice + Memory

Month 2-3
  • Web Speech API integration
  • Real-time voice waveform visualiser
  • Per-user memory (pre-fill returning users)
  • Agent profile + response via conversation
  • Language auto-detection (EN/DE)

Phase 3 — Intelligence

Month 4+
  • Cross-platform learning (optimal question order)
  • Market-specific terminology adaptation
  • Outcome-based flow optimisation
  • Contextual help within conversation
  • Multi-turn clarification for ambiguous answers

Ready to build?

Phase 1 ships as part of Sprint Zero. Text conversation first, voice follows.
