- **How to make portals “zero manual” from Yalla**
    
    Great question. Short version: full automation is doable, but each portal has its own gate + tech format. Here’s the practical breakdown and the exact switches we can give to Owners/Agents in Yalla.
    
    # **How to make portals “zero manual” from Yalla**
    
    ## **Common setup you’ll need once (applies to all portals)**
    
    - A **commercial contract** with the portal (as an advertiser/agent or as a certified “software house”/feed provider on behalf of agencies).
    - **Branch/office IDs** per agency, and **feed credentials** (API/FTP, sometimes client TLS certs).
    - A small middleware that maps your Airtable/DB fields → each portal’s schema (RTDF/BLM), then posts updates on create/edit/sold.
    
    After that, Agents can toggle per listing: **“Send to Rightmove / Zoopla / OTM”** and we push in real time or on schedule.
    
    ---
    
    ## **1) Rightmove**
    
    **How it works (2 supported feeds):**
    
    - **RTDF (Real-Time Data Feed)** – JSON web service, push adds/updates/deletes in real time. Requires being whitelisted as a data-feed provider and test certification.
    - **ADF/BLM (Automated Data Feed)** – legacy file (BLM) via FTP; still supported, but slower (batch). Rightmove hosts specs + test process.
    
    **What you need:**
    
    - Rightmove to set you up as a **software house/data feed provider** and issue **test + live credentials** (network/branch auth).
    - Each **agency’s branch ID** tied to your feed.
    - Field mapping to their schema (price, address, media, EPC, status codes, etc.).
    
    **Leads back into Yalla:**
    
    - Rightmove typically sends **ADF XML leads** (email) or via partner lead endpoints; you can parse/poll and drop into Airtable. (Rightmove public docs emphasise ADF; details provided during onboarding).
    
    **Owner/Agent toggle in Yalla:**
    
    - Per listing: **[✓] Publish to Rightmove** (sends RTDF add/update; uncheck sends remove).
    - Agency settings: paste **Branch ID + credentials** once; we handle the rest.
    
    ---
    
    ## **2) Zoopla**
    
    **How it works:**
    
    - Supports **Real-Time feed** via their partner program (the public Listings API isn’t open; you must engage commercially).
    - Many setups still use **BLM over FTP** via a CRM/software house. Zoopla’s own help guides explain feed vs manual, and “Real-time service.”
    
    **What you need:**
    
    - **Commercial access** via Zoopla/Alto; they’ll issue **branch codes + feed credentials**.
    - For Real-Time, expect **client certificates/TLS** and strict schema. (Vendor docs + Property Hive guidance reference certificates & portal settings.)
    
    **Leads back into Yalla:**
    
    - Zoopla provides **lead push/poll** mechanisms with defined fields; you can ingest to Airtable.
    
    **Owner/Agent toggle in Yalla:**
    
    - Per listing: **[✓] Publish to Zoopla**.
    - Agency settings: **Branch code + certs/FTP** added once.
    
    ---
    
    ## **3) OnTheMarket (OTM)**
    
    **How it works:**
    
    - Offers a **Real-Time feed** with an API **modelled on Rightmove RTDF**; also supports BLM. (RTDF spec docs mirror Rightmove’s structure.)
    
    **What you need:**
    
    - **Contract + branch ID**, and **RTDF credentials** from OTM.
    - Same mapping approach as Rightmove (status, media, features).
    
    **Leads back into Yalla:**
    
    - Typically email/ADF or partner endpoints; parse into Airtable (similar to others). (Vendors like Property Hive note real-time and enquiry import options.)
    
    **Owner/Agent toggle in Yalla:**
    
    - Per listing: **[✓] Publish to OnTheMarket** with “Only With Us” exclusivity window if desired.
    
    ---
    
    ## **Formats you’ll support in the middleware**
    
    - **RTDF (real-time JSON over HTTPS)** – Rightmove/OTM/Zoopla variants.
    - **BLM + media zip over FTP** – legacy but widely accepted; good fallback.
    
    ---
    
    ## **“Can the Owner (private seller) just allow this in settings?”**
    
    - **Rightmove/Zoopla/OTM do not accept private FSBO direct** — they require an **agency/advertiser contract**.
    - For FSBO via NoAgent, you either:
        1. **List under your own advertiser account** (you’re the “agent of record” for the portal add-on), or
        2. **Partner with an aggregator/agent** who pushes on your behalf.
    
    Agents using **Yalla** can connect their own portal credentials once, then each listing can be toggled on/off per portal by the **Agent** (and optionally allowed by the **Owner** via a checkbox). The **Owner** alone can’t publish to RM/Zoopla directly without an agent/advertiser entity.
    
    ---
    
    ## **What you’ll build in Yalla settings (practical)**
    
    - **Agency → Portals** (section):
        - Rightmove: Branch ID, RTDF credentials (or FTP).
        - Zoopla: Branch code, RTDF certs (or FTP).
        - OTM: Branch ID, RTDF credentials (or FTP).
        - Test buttons: *Send sample listing*, *Remove*, *Get branch inventory*. (Rightmove expects these calls in certification.)
    - **Listing → Distribution** (panel):
        - Toggles: **Rightmove / Zoopla / OTM**
        - **Exclusivity** (OTM “Only With Us” 24/48/72h option).
    - **Leads → Ingest**:
        - Parse **ADF XML / JSON** from emails or API into Airtable (Zoopla lead field defs are public).
    
    ---
    
    ## **Gotchas (so you’re not surprised)**
    
    - **Certification/whitelisting**: Rightmove RTDF requires a short certification with their team before going live.
    - **Media rules**: image sizes, count, and order matter; failing rules can silently reject. (Follow the feed specs.)
    - **Status mapping**: “For Sale / Under Offer / SSTC / Withdrawn / Sold” must map 1:1 to each portal’s enums.
    - **Zoopla access**: listings API isn’t public; you need a **commercial arrangement** (or use BLM via a software house).

    3. Add **Zoopla** via **BLM** first, then upgrade to **RTDF** once commercial access is granted.

## Running Costs

- **1) Portal contracts (what agents/branches pay)**
    - **Rightmove:** benchmark **~£1,524 / advertiser / month** (estate agency ARPA ~£1,440). Exact fees vary by package/area, but this is the best public proxy.
    - **Zoopla:** industry estimate **~£350–£400 / branch / month** (≈¼ of RM’s ARPA). Exact pricing is quoted per account.
    - **OnTheMarket:** positions itself as **“a fraction of competitors”**; pricing is quote-based and generally below Zoopla for many agents.
    
    > For FSBO: portals don’t take private sellers directly — listings must run under an advertiser/agent contract.
    > 
    
    # **2) Feed / integration costs (to make it hands-off)**
    
    - **Rightmove**: Real-Time Data Feed (RTDF) or legacy **BLM/FTP**. No public “feed fee”, but you must be onboarded/certified and have a live advertiser contract.
    - **Zoopla & OTM**: both support **real-time feeds** (Zoopla uses its own RT format; OTM mirrors RM’s RTDF). Again, commercial access + credentials required.
    
    # **3) Ops stack you’ll likely pay for (platform side)**
    
    - **Framer** (marketing site): **$10 / $30 / $100** mo (Basic/Pro/Scale). CMS/page/bandwidth limits + add-ons apply.
    - **Airtable** (CRM/data): **$20 / seat / mo (Team)**, **$45 / seat / mo (Business, annual)**.
    - **Make.com** (automations): from **$9 / mo** for Core.
    - **Email** (lead parsing/notifications): Twilio **SendGrid** paid plans from **$19.95 / mo**; free plan is being retired post-trial.
    - **Media CDN** (photos/video): **Cloudinary** has a generous **Free** tier; paid from **$89 / mo** if you grow.
    - **Domain**: **.house** typically **~$13+ / yr** first year (renewals vary by registrar).
    - **Optional messaging (WhatsApp)**: Twilio adds **$0.005 / msg** on top of **Meta’s** per-template fees (new per-message pricing from **1 July 2025**). Budget a small variable cost if you enable WhatsApp.
    
    # **4) Per-listing services you might pass through (or upsell)**
    
    - **EPC** (seller requirement): **~£60–£120** typical.
    - **Photography**: **~£100–£200+** per property depending on package/region.
    - **Floorplan**: **~£75–£150** common range (varies widely).
    
    # **5) Compliance**
    
    - **ICO data-protection fee (UK)**: most small orgs pay **£52 or £78 / year** (tiered by size/turnover).
    
    ---
    
    ## **What this means for Yalla.House / NoAgent**
    
    - If an **agent connects their own portal creds**, your feed pushes are “free” from the portal’s side; their monthly **portal contracts** are the big ticket.
    - If **you** list FSBO on portals under **your** advertiser account, you’ll carry those monthly portal fees — so treat it as a pass-through add-on.
    - Your **platform** costs (Framer/Airtable/Make/Email/CDN/Domain/ICO) are modest and predictable; scale mostly affects media and email/messaging.
