# RoadMap

# **WHO IS YOUR CUSTOMER?**

## **1) Market Segmentation**

**Primary segments (your core loop)**

- **Owners (Sell/Rent)** — want speed + control + less admin, may still use an agent.
- **Home-Hunters (Buy/Rent)** — want booking + answers + less spam.
- **Real Agents** — want fewer repetitive calls/emails + faster conversion + cleaner listing data.
- **Service Providers** (photographers, EPC, surveyors, notaries, trades) — want steady jobs + instant evidence upload.
- **Platforms / Country partners** (portals, feed partners, “agent-of-record”) — distribution/compliance layer (later).

**Segment logic (why this segmentation works)**

- Owners + Providers **create supply**
- Agents **multiply conversion + distribution**
- Home-Hunters **create demand + liquidity**
- Platforms **unlock market access** (country-specific)

## **3) Build an End User Profile (End user ≠ payer)**

**End users who touch the product weekly**

- **Owners**: upload docs, answer Q&A once, set slots, approve viewings/offers.
- **Home-Hunters**: request viewing, verify, ask questions, offer/application.
- **Agents**: receive pack link, quote, handle enquiries, run viewings, manage follow-up.
- **Providers**: accept jobs, upload evidence, close task.

**Critical behavior you’re replacing**

- Phone tag + reconfirming + repeated basic questions + owners being the “real source of truth”.

## **4) Calculate the TAM Size for the Beachhead Market (Berlin)**

You can treat TAM in 3 layers: **transaction-driven (sell), turnover-driven (rent), services-driven (providers).**

**Anchor datapoint to size beachhead activity**

- A Berlin residential market report notes ~**11,300 apartments notarized by end of 2025** (apartment sales).

**TAM model (simple, directional)**

- **Sell-side TAM (Berlin apartments)**: ~11.3k sales/year × (average agent fee + add-ons you can influence).
- **Rent-side TAM**: larger volume than sales (turnover is high in a tight market), but varies by submarket and regulation; treat as “bigger but noisier.”
- **Services TAM**: photos/floorplan/EPC/surveys are attachable to both sell + rent flows.

**What to put in Notion as the actual calculation**

- 3 scenarios (Conservative / Base / Aggressive):
    - Adoption (1%, 3%, 7% of relevant listings)
    - ARPA (average revenue per актив listing: package + add-ons + referral)
    - Contribution margin assumptions

# **HOW DOES YOUR CUSTOMER ACQUIRE YOUR PRODUCT?**

## **12) Determine the Customer’s Decision-Making Unit (DMU)**

**Owners (micro DMU)**

- Decision: owner + partner/spouse; sometimes lawyer/notary influence.

**Agents (small org DMU)**

- Branch manager + admin staff (the people who feel the pain)
- Sometimes owner/director signs off.

**Providers**

- Owner/operator decides fast if jobs are real.

**Platforms**

- BD + compliance/legal.

---

## **13) Map the Process to Acquire a Paying Customer**

**Owner-paying path (fastest)**

Entry → Create Pack → Add service (photos/EPC/floorplan) or pay for “distribution add-on” → convert.

**Agent-paying path (later)**

Owner sends pack link → agent quotes inside NoAgent → agent sees time saved → pays for “territory/lead priority” or “agent seat” once value is proven.

**Provider-paying path (optional)**

Usually you take a margin/fee per job or subscription for preferred placement.

---

## **14) Map the Sales Process to Acquire a Customer**

**Berlin GTM wedge (non-SEO, multiplier-led)**

1. Seed 10–20 real listings (friends/network)
2. Those owners message 30–60 agents **through NoAgent** (“quote me using this pack”)
3. Agents adopt the scheduling link + comms (because it reduces daily pain)
4. Agents push more owners into NoAgent (“use this link so I can act fast”)
5. Providers get jobs through the checklist → proof auto-uploads → pack quality rises → more agent adoption

# **HOW DO YOU MAKE MONEY OFF YOUR PRODUCT?**

## **15) Design a Business Model**

**Phase 1 (Owner-led revenue)**

- Listing package (free/basic + paid upgrades)
- Add-ons: pro photos, floorplan, EPC/survey, signboards
- Paid distribution/portal listing where legally possible (or via partner/agent)

**Phase 2 (Agent-led multiplier)**

- Subscription per territory / lead priority / “agent seat”
- Success fee on completed deal (only where legal/viable)
- Syndication/export features (workflow becomes sticky)
